# 创造型模式

## 单例模式

要求

1. 某个类只能有一个实例
2. 必须自行创建实例
3. 必须自行向整个系统提供这个实例

实现

1. 只提供私有的构造方法
2. 含有一个该类的静态私有对象
3. 提供一个静态的公有方法用于创建、获取静态私有对象

使用场景：

1. 创建对象时占用资源过多，但同时又需要用到该类对象
2. 对系统内资源要求统一读写，如读写配置信息
3. 当多个实例存在可能引起程序逻辑错误，如号码生成器

懒汉式与饿汉式

懒汉式：空间换时间
饿汉式：时间换空间

饿汉式线程安全
懒汉式存在线程风险：1.同步锁；2.双重校验锁；3.静态内部类；4.枚举
